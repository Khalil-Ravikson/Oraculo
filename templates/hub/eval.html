Como Organizar os Metadados das Ingestões (Taxonomia UEMA)

Para obter a máxima precisão RAG e evitar contaminação de contexto, cada chunk armazenado no Redis deve conter metadados taxonômicos indexados. Organize as tags no momento da ingestão (via LlamaParse ou scripts de carga):

1. eixo: Classifica o tema principal (ex: Institucional, Ensino, Pesquisa, Extensao, Sistemas).
2. setor: Define a pró-reitoria ou diretoria dona do dado (ex: PROG, PROEXAE, PPG, CTIC, Reitoria, Geral).
3. tipo_doc: Tipo de arquivo (ex: Edital, Calendario, Resolucao, Manual, Contatos).
4. ano: Ano letivo do documento (ex: 2025, 2026). Essencial para barrar dados antigos de calendários obsoletos.
5. campus: O campus aplicável (ex: Sao_Luis, Caxias, Imperatriz, Todos).

Durante a busca, o QueryTransformService lê o histórico do chat (ex: "onde fica?") e reescreve a query anexando filtros automáticos baseados nessa taxonomia (ex: @ano:{2026} @tipo_doc:{Calendario}).

Ingestão Automática via LlamaParse

O LlamaParse converte PDFs complexos em Markdown estruturado, ideal para extrair tabelas de calendários acadêmicos e editais de vestibulares. Os arquivos colocados em dados/tmp/martech_eval_llm_document.pdf e no Calendário podem ser processados com o pipeline local.

# Exemplo de comando Python para Ingestão via LlamaParse:
from src.rag.ingestion.pipeline import IngestionPipeline
from src.infrastructure.settings import settings

# Configura a API Key no .env
settings.LLAMA_CLOUD_API_KEY = "sua_chave_aqui"

pipeline = IngestionPipeline.build_auto(
    file_path="dados/tmp/martech_eval_llm_document.pdf",
    doc_type="edital"
)
# Executa síncrono ou assíncrono
result = await pipeline.run_async(
    file_path="dados/tmp/martech_eval_llm_document.pdf",
    doc_type="edital",
    label="MARTECH EVAL"
)
print(f"Sucesso: {result.success} | Chunks salvos: {result.chunks_saved}")
