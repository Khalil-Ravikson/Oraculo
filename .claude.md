# Contexto do Projeto Oráculo
Este é um agente de IA corporativo e assistente acadêmico.
Stack principal: Python 3.12.8, FastAPI, LangChain/LlamaIndex, Celery, Alembic e Redis.

# Regras de Desenvolvimento
- **Tipagem:** Use SEMPRE type hints estáticos em todas as funções e métodos.
- **Assincronismo:** Priorize `async/await` para I/O e chamadas de banco.
- **Economia:** Antes de gerar blocos massivos de código, apresente o raciocínio.
- **Aprendizado:** Se eu corrigir um padrão arquitetural ou um erro recorrente seu, atualize este arquivo `.claude.md` automaticamente para incluir a nova regra.
- Vá direto ao ponto, mostre o código sem pedir desculpas.
- **Sem LangGraph:** LangChain fica só para embeddings. `StateGraph`/`checkpointer`/roteamento via grafo do LangGraph estão descartados como runtime — já testado, não funcionou (travou em `state`/`builder`). Roteamento e HITL continuam 100% custom (Router/Supervisor + Celery/Redis). Nunca sugerir migrar `router/`, `application/runtime/dispatcher.py` ou `agents/sigaa/auth_flow.py` para o grafo do LangGraph, mesmo que uma referência externa recomende.

### Aprovados (uso pontual e restrito ao escopo indicado — não copiar arquitetura completa)

- **[vvandk/kinit](https://github.com/vvandk/kinit)** — só o desenho de RBAC (menu dinâmico + botão + role-departamento) e o padrão JWT/OAuth2 via `Depends()`. Não usa Celery (usa APScheduler+MongoDB) — ignorar o resto.
- **[deepwiki: lijianqiao/fastapi-admin-v3 RBAC](https://deepwiki.com/lijianqiao/fastapi-admin-v3/3.2-rbac-and-permission-system)** — permission string `módulo:recurso:ação` + dependency factory `check_perm()`, stack mais próxima da nossa que o kinit. Repo pequeno (6★) e fonte é wiki gerada por IA — conferir código real antes de portar.
- **[codingeasypeasy — RBAC dinâmico em FastAPI](https://codingeasypeasy.com/blog/role-based-access-control-rbac-with-dynamic-permissions-in-fastapi-a-comprehensive-guide/)** — permissão condicional via `ast.parse()` (não `eval()`). Só vale a pena se surgir necessidade real de regra por `centro`/`campus`/`setor` além do enum de role atual.
- **[evolution-foundation/evolution-api](https://github.com/evolution-foundation/evolution-api)** — é Node.js/TS, não FastAPI (é o gateway que já rodamos, não um padrão de código a copiar). Usar só a doc de eventos de webhook para checar cobertura em `webhook_controller.py`.
- **[Artigo Jellyfish — LLM-Driven API Orchestration](https://www.jellyfishtechnologies.com/llm-driven-api-orchestration-using-langchain-celery-redis-queue/)** — valida o padrão `FastAPI → Redis Queue → Celery → DB` via `.delay()`. Não adotar o agente LangChain do artigo como decisor de rota.
- **[celery/celery — next-steps.rst](https://github.com/celery/celery/blob/main/docs/getting-started/next-steps.rst)** — doc oficial atual. Ver Canvas (`chain`/`group`/`chord`) como possível simplificação do `dispatcher.dispatch()`.
- **[SteliosGian/fastapi-celery-redis-flower](https://github.com/SteliosGian/fastapi-celery-redis-flower)** — ideia concreta: adicionar Flower ao `docker-compose.yml` para monitorar as 5 filas Celery em tempo real (hoje só Prometheus/Grafana).
- **[Darshangoswami07/agentic-ai-research-platform](https://github.com/Darshangoswami07/agentic-ai-research-platform)** — stack mais próxima da nossa (FastAPI+Celery+Postgres+Redis+RBAC). Repo com 0★, README "enterprise-grade" de projeto pessoal — usar só como inspiração de nomenclatura/estrutura de agentes (Planner/Researcher/Analyst/Critic), nunca como fonte validada. Não portar Qdrant nem LangGraph.
- **[Shy2593666979/AgentChat](https://github.com/Shy2593666979/AgentChat)** — repo com tração real (810★). Útil só como benchmark de nomenclatura de memória em camadas. Usa MySQL/ChromaDB/Vue — nada de infra portável.
- **[note.com — HITL com LangGraph + FastAPI](https://note.com/nzw_/n/n4095231ae48d?hl=en)** — único princípio realmente agnóstico de framework do lote de HITL: separar o nó de aprovação do nó de efeito colateral (idempotência). Conferir se `auth_flow.py` já respeita isso.
- **[LangChain Docs — Multi-agent](https://docs.langchain.com/oss/python/langchain/multi-agent)** — só vocabulário/taxonomia de padrões (router, handoffs, subagents). Nunca adotar o runtime `StateGraph` (ver regra "Sem LangGraph" acima).
